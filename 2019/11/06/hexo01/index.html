hexo+GitHub搭建博客

Author: Seven Date: 十一月 6, 2019  10:37:23

一、安装NODE和GIT

1.node下载网址为https://nodejs.org/en/download/
git下载网址为：https://git-scm.com/download/win

2.根据本机类型，选择相应的版本下载

3.node一路默认安装即可 Git有两个页面要注意选择其余的部分默认安装
注：nodejs默认的安装中会包含环境变量的配置和npm工具的安装。

4.检验是否安装成功：在“开始”菜单中输入cmd，打开命令行窗口
（1）输入 node –v，若显示版本号，则nodejs安装成功；
（2）输入 npm -v，若显示版本号，则npm安装成功；
(3)

至此，Hexo的环境配置已经全部完成。

二、Hexo的安装

1.在CMD窗口下使用 npm 命令安装 Hexo 客户端，输入：npm install -g hexo-cli （安装完末尾出现hexo版本号则表示安装成功）

2.在电脑中新建一个文件夹用来保存我们的博客 名字和路径最好都是英文

3.在CMD窗口下用命令切换到这个文件夹

（比如我的就是先输入D:切换到D盘 然后输入cd blog切换到D盘的blog文件夹）

4.切换到博客文件夹后 输入hexo init 初始化hexo

（出现 INFO Start blogging with Hexo!字样表示安装成功）

5.生成 public 静态资源:输入hexo g

6.启动本地服务器:输入hexo s

（这时可以登陆网址localhost：4000查看本地博客）

三、开写第一篇博客文章

1.上述网站中出现的Hello World文章其实在blog\source_posts文件目录下默认创建的，因此我们可以直接在该文件目录下创建自己的博客内容。

2.新建第一篇博客文件
在blog文件下使用命令：hexo new 文件名。这样hexo就会帮我们在blog\source_posts下自动生成制定名称的文件,后缀名为.md。用这个命令的好处是帮我们自动生成了时间。

3.打开、撰写、保存新文件： 可以使用txt操作，打开文件后输入以下内容 hello 你好啊，并以UTF-8格式来保存。

（这时刷新刚刚的本地博客我们可以发现我们刚刚写的东西出现在了博客上）

四、GitHub和本地hexo博客连接

1.注册一个GitHub账号

2.创建一个新的仓库：登录官网后，点击标签后输入用户名、密码。然后创建一个新的仓库

（完成后，可以点击该仓库，进入仓库的配置界面）

3.配置SSH key

打开git bash
输入 cd~/.ssh（这一步用来测试本机是否存在ssh key）
如果不存在那就需要我们生成一个ssh key
输入 ssh-keygen-t rsa -C "github邮箱名称"
（这一条指令用于生成ssh key）
然后连续3次回车，最终会生成一个文件在用户目录下
打开用户目录，找到.ssh\id_rsa.pub文件，
记事本打开并复制里面的内容，打开你的github主页，
进入个人设置 -> SSH and GPG keys -> New SSH key：
把刚刚生成的ssh key添加进去
测试本机是否可以通过ssh连接到自己的github仓库。
输入：ssh -T git@github.com 注意邮箱地址不用改
然后输入yes 看到You've successfully字样说明SSH已经配置成功

4.修改blog\ _config.yml文件，将最后一部分修改为如下内容

deploy:  
type: git
repo: git@github.com:用户名/用户名.github.io.git
（repo后面的内容拷贝自你自己创建的GitHub仓库）
branch: master

5.安装一个用于关联hexo和git的小插件

在CMD窗口下输入npm install hexo-deployer-git –save

6.将hexo部署至自己的github仓库上：
在CMD窗口下 切换到博客文件夹 输入 hexo d 指令
注：d-deoly。

根据错误提示消息，输入两条git的配置指令：
用户邮箱和用户名报错 把那两条复制下来 把邮箱和名字改成自己的

7、在浏览器中输入我们的仓库地址，发现网页更新为本地的静态网页，同时仓库中多了很多文件。但这时候发现只有我们的第一篇默认的hello world博文！因为我们新建的那篇博文没有使用命令生成静态资源！

8.在git bash窗口下输入git init

这样就能把它变成一个git仓库，这时文件夹内会多出一个.git目录。

9.在CMD窗口下 切换到博客文件夹
输入① hexo g ② hexo d

第一条指令的作用是更新public下的index文件。

至此，博客就已经搭建完毕

over.
